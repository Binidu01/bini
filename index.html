Getting Started

What is Bini.js? +

Bini.js is a next-generation React framework built for performance, security, and developer experience. It combines the best features of modern frameworks with built-in source protection, lightning-fast Vite-based builds, and zero-config setup. Perfect for production-grade applications requiring both speed and code security.

How do I install Bini.js? +

Create a new Bini.js project using: npx create-bini-app@latest my-app

This sets up a complete project with TypeScript, TailwindCSS, and all dependencies. Alternatively, clone the starter template from GitHub.

Is Bini.js production-ready? +

Absolutely! Bini.js includes production-optimized builds with code splitting, built-in source code obfuscation and protection, SEO-optimized server-side rendering, advanced caching strategies, and error boundaries. It's already used by companies in production environments.

Development

How do I create a new page? +

Bini.js uses file-based routing. Create files in src/app/ directory:

  • src/app/about/page.tsx ‚Üí /about
  • src/app/blog/[slug]/page.tsx ‚Üí /blog/any-slug
  • src/app/dashboard/layout.tsx ‚Üí Layout for pages

Each page file should export a default React component.

Can I customize TailwindCSS? +

Yes! Bini.js comes with TailwindCSS fully configured. You can:

  • Modify tailwind.config.js for custom themes
  • Add custom colors, fonts, and spacing
  • Extend with plugins like @tailwindcss/forms
  • Use CSS modules alongside Tailwind
  • Configure dark mode with class or media strategy
Does Bini.js support dark mode? +

Yes! Bini.js has first-class dark mode support. Use Tailwind's dark: prefix for styles, toggle between modes with context, system preference detection, persistent theme selection, and smooth transitions.

API & Backend

How do I add API routes? +

Create files in the src/api/ directory:

  • src/api/users.js ‚Üí /api/users
  • src/api/posts/[id].js ‚Üí /api/posts/123

Each API handler can return JSON directly. No need for manual response objects.

Can I use databases with Bini.js? +

Yes! Bini.js supports any database:

  • MongoDB with Mongoose
  • PostgreSQL with Prisma
  • Firebase and Supabase
  • REST APIs and GraphQL
  • Local storage and cookies
  • React Query for server state
  • Zustand or Redux for client state

Production & Styling

How do I build for production? +

Use the built-in commands:

npm run build # Build for production
npm run start # Preview build
npm run dev # Dev server
npm run analyze # Bundle analysis

The build output is optimized and ready for Vercel, Netlify, or any platform.

Where can I find examples? +

Check the official examples at github.com/Binidu01/bini-examples

Includes: E-commerce stores, Blog platforms, Dashboard UIs, Authentication flows, API integrations, and Real-time applications.

How do I change site metadata? +

Edit the metadata object in src/app/layout.tsx:

export const metadata = {
  title: 'Your Site',
  description: 'Description',
  keywords: ['kw1', 'kw2'
}

You can also set per-page metadata.

Security & Troubleshooting

How does source protection work? +

Bini.js uses advanced obfuscation techniques:

  • Code minification and mangling
  • String encryption
  • Control flow flattening
  • Dead code injection
  • Anti-debugging protections

This makes reverse engineering extremely difficult while maintaining runtime performance.

What makes Bini.js so fast? +

Bini.js achieves exceptional performance through:

  • Vite-based development with instant HMR
  • Optimized production builds with tree shaking
  • Intelligent code splitting
  • Built-in image optimization
  • Lazy loading components
  • Minimal runtime overhead
  • Efficient hydration strategies
How do I debug issues? +

Bini.js provides multiple debugging tools:

  • Detailed error overlays in development
  • Source maps for production errors
  • Performance profiling tools
  • Network request monitoring
  • Console logging with source locations
  • Error boundary components
  • TypeScript for compile-time checks
